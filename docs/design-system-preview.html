Design System · v1

Healthcare commerce, built for trust.

A premium storefront for medicines, lab tests, and health packages in Pakistan. White ground, blue for the one action that matters, green reserved for things genuinely verified. Designed mobile-first for mid-range Android on 4G — which is where this product actually lives.

Trust over delight

People here are often anxious, and often buying for someone else. Legibility beats cleverness. Nothing decorative competes with anything safety-critical.

Apple shell, Amazon decision

Restraint and whitespace for navigation and marketing. Density where people decide — price, stock, and delivery date visible without scrolling.

One blue action per view

Blue is scarce on purpose. If three things on a screen are blue, none of them answers “what do I do next?”

Color

Measured, not estimated

Every value is checked against white. The pairs exist because a single hue can rarely serve both a fill and body text — green-600 fails AA at 3.38:1, so it fills badges while green-700 does the writing. Collapsing those into one green is the most common accessibility failure in “green means good” systems.

Amber, not red, marks prescription-required. It is a condition of purchase, not an error — and it gets a reserved hue so it is never mistaken for either a warning or a success. Neutrals are slate, carrying a slight blue bias so greys sit with the accent instead of muddying against it.

Typography

Inter Variable, self-hosted

Self-hosted because a font CDN round-trip costs roughly 300ms of LCP on 4G. Body text never drops below 16px — it is the legibility floor, and it also stops iOS Safari zooming when an input takes focus. Tabular numerals are mandatory on every price, quantity, and lab value, so totals do not shift while they update.

Radius & elevation

Borders, not shadow stacks

Radius scales with element size — 12px on a 400px card reads tight, 16px on a 32px badge reads like a pill. Consistency here is proportional, not literal. Surfaces are defined by a 1px border by default; heavy shadow is the fastest way to look dated, so elevation is reserved for things that genuinely float.

Product card

Fixed ratio, clamped title

A fixed 1:1 image and a two-line title clamp keep every card the same height, which is what stops the grid reflowing as images load — the main source of layout shift on listing pages. The generic name sits under the trade name because many customers arrive holding a doctor's note that lists a molecule, not a brand.

Loading, empty, error

Designed, not defaulted

Skeletons match the final layout's dimensions, so nothing jumps when content arrives — a centred spinner gives no layout hint and reads slower even when it isn't. Empty states always carry a way forward; “No results” on its own is a dead end, not a design.

The three that decide conversion

Product detail, checkout, and the pharmacist queue. The first two carry the storefront's revenue; the third sits on the critical path of every single prescription order, which makes it the highest-leverage screen in the admin panel.

The admin panel is the one place that drops the white ground. Staff sit here for hours, so density is a kindness — the same density on the storefront would read as noise. Expiry status carries a severity stripe as well as a word, so a shelf problem is visible before anyone reads a row.
